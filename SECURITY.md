# Security Policy

## Supported Versions

This project is actively maintained. Only the latest major version is supported. Older versions may not receive security updates.

## Reporting a Vulnerability

If you discover a security vulnerability, please follow these steps:

1. **Do not disclose publicly.**
2. Email the maintainer: [ashusriwastav07@gmail.com](mailto:ashusriwastav07@gmail.com)
3. Include a detailed description, steps to reproduce, and potential impact.
4. You will receive a response within 48 hours.

## Responsible Disclosure

We appreciate responsible disclosure. All valid reports will be investigated and acknowledged. Critical vulnerabilities will be patched as soon as possible.

## Dependencies

- All dependencies are kept up to date. Please report any known vulnerabilities in third-party packages.

## Supabase & GitHub API

- User data is stored securely in Supabase.
- No sensitive information is exposed in the frontend.
- GitHub API tokens are rotated and never stored in client code.
